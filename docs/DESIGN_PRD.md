# 119덱 — 디자인 PRD (클로드 디자인 브리프)

> **한 줄**: 소방공무원 수험생이 자투리 시간에 심화·OX 기출을 풀고, 틀린 걸 바로 알고, 약점을 반복하는 **모바일 문제풀이 앱**. 이미 동작하는 Flutter MVP가 있고, 이제 **화면 비주얼 디테일을 확정**하려 한다.
> **당신(디자이너)에게 원하는 것**: 아래 화면들의 ①고품질 **모바일 목업(HTML/CSS)** + ②**비주얼 스펙 문서**. 현재 디자인 시스템은 *출발점*이며, 더 나은 방향이면 **토큰(색·타이포·여백)까지 재제안 환영**.

---

## 0. 이 문서 사용법 (디자이너용)

- 이건 "무엇을·왜"의 브리프다. 픽셀은 당신이 정한다.
- **자유도**: 비주얼 탐색 허용. 현 디자인(§4)은 baseline일 뿐, 무드보드부터 다시 제안해도 된다. 단 §5 **하드 제약**은 반드시 지킨다.
- **산출물 2종**(§7): (A) 화면별 인터랙티브 HTML 목업, (B) 비주얼 스펙 문서. 목업엔 **§8의 진짜 문항 카피**를 써라(lorem 금지).
- **최종 구현은 Flutter(Material 3, 외부 UI 의존성 0)**. 즉 순수 위젯으로 재현 가능한 범위여야 한다(§5). 화려한 커스텀 그래픽·불가능한 블러 남발 금지.

## 1. 제품 · 사용자 컨텍스트

- **사용자**: 소방공무원 공채 준비생. 헤비유저는 하루 50~100문제. **장시간·반복** 문제풀이가 핵심 사용 맥락.
- **핵심 JTBD**: "자투리·집중 시간에 실전 난이도 문제를 풀고, 틀린 걸 바로 알고, 약점을 반복해 시험을 통과하고 싶다."
- **파생**: 시험 직전 실전 시뮬레이션(전체 25문 일괄채점), 오답만 골라 빠른 복습.
- **차별화 = 콘텐츠 깊이**(심화·교차법령·OX). 기능 수로 경쟁 안 함 → **UI는 조용하고, 콘텐츠가 주인공**이어야 한다.
- **감성 톤 목표**: 신뢰감 있고 차분한 "공부 도구". 게임처럼 요란하지 않게, 그러나 **정답/오답 피드백은 명확하고 시원하게**. 소방 정체성(레드)은 절제된 액센트로.

## 2. 화면 인벤토리 (5화면 + 상태)

각 화면은 **로딩/빈/에러/데이터** 상태를 가진다. 목업은 데이터 상태 위주 + 핵심 빈/결과 상태를 포함하라.

| # | 화면 | 목적 | 핵심 요소 | 꼭 그려야 할 상태 |
|---|---|---|---|---|
| 1 | **홈** `/` | 과목·모드 선택 진입 | 과목 카드(소방관계법규) · 모드 4종(전체/랜덤/오답/시험) · **이어풀기** 강조 진입점(세션 있을 때) · 우상단 설정 아이콘 | 데이터 · (이어풀기 有/無) |
| 2 | **풀이** `/quiz` | normal·random·review 공용 핵심 루프 | 상단 진행바(N/총) · 문항 카드(지문+유형배지) · 선택지(①~④ 또는 O/X) · **선택 즉시 채점**(색+아이콘) · 해설 카드 · 하단 "다음"/"결과 보기" | 문항(채점 전) · 채점 후(정답/오답) · 오답세트 빈 상태 |
| 3 | **시험** `/exam` | 전체 25문 실전, 즉시채점 숨김 | 진행바 · 문항 카드 · 선택(채점 표시 없음, 재선택 가능) · "다음"/마지막에 "제출" | 문항 · 제출 직전 |
| 4 | **결과** | 점수 + 오답 리뷰 | 큰 점수(정답수/총, 정답률%) · **오답 리뷰 리스트**(문항·내 답✕·정답✓·해설) · "다시 풀기"/"홈으로" | 만점 · 오답 있음 |
| 5 | **설정** `/settings` | 테마·앱 정보 | 테마 라디오(시스템/라이트/다크) · 앱 정보 | 기본 |

## 3. 핵심 인터랙션 · 상태 전이 (여기가 디테일의 승부처)

### 3.1 풀이 루프 (앱의 심장 — 가장 공들여야 함)
1. 문항 표시(지문 = 화면 주인공, 큰 글씨·넉넉한 행간).
2. 선택지 탭 → **즉시 채점**:
   - 내가 고른 게 정답 → 초록 + ✓
   - 내가 고른 게 오답 → 빨강 + ✕ **동시에 정답 선택지를 초록 ✓로 함께 표시**(뭐가 맞는지 즉시 학습).
   - 채점 후 모든 선택지 비활성.
3. **해설 카드** 등장(정답/오답 배너 + 해설 본문).
4. 하단 버튼 "다음" → 다음 문항 / 마지막이면 "결과 보기".
5. 오답은 **자동으로 오답세트에 누적**(버튼 없음). 오답 재풀이(review 모드)에서 맞히면 **즉시 제거**.
> **디테일 요청**: 채점 순간의 색 전환·아이콘 등장을 어떻게 "명확하지만 요란하지 않게" 보여줄지가 핵심. 정답 초록과 오답 빨강, 그리고 브랜드 레드(버튼)가 **서로 혼동되지 않게** 톤·형태로 구분하는 설계를 제안하라.

### 3.2 OX 문항
- 선택지가 O/X 2개. 객관식(①~④)과 **같은 ChoiceTile 언어**로 보이되, O/X는 더 크고 시원한 2-버튼 레이아웃도 제안 가능.

### 3.3 시험 모드
- 풀이 뷰 재사용하되 **채점 색·해설 숨김**. 선택은 "내가 이걸 골랐다"만 표시(중립 강조), 재선택 허용.
- 마지막 문항에서 버튼이 "제출"로 바뀜 → 일괄채점 → 결과.

### 3.4 결과
- **점수를 크게**(정답수/총 + 정답률%). 만점과 오답 있음의 감정 톤을 다르게(과하지 않게).
- 오답 리뷰: 문항별 카드에 지문·내 답(✕)·정답(✓)·해설. 스크롤 리스트.

### 3.5 이어풀기 (normal 한정)
- 홈에서 세션 있으면 **상단 강조 타일**("정규 · N/총 이어풀기"). "전체 풀이"(처음부터)와 시각적으로 구분.

## 4. 현재 디자인 baseline (참고 — 존중이 아니라 출발점)

> 이미 구현돼 동작한다. 이 방향이 좋으면 다듬고, 아니면 **대안을 제안**하라. 아래는 "지금 이렇다"의 스냅샷.

- **방향**: 차분한 뉴트럴 표면 + **소방 레드(#D32F2F) 액센트**(상단바·진행바·주요 버튼·선택 링에만). 정답=초록(#2E7D32) / 오답=빨강(#C62828), **항상 아이콘 동반**.
- **중립 표면(살짝 웜)**: bg #FAFAF9 / surface #FFFFFF / outline #E4E2DE / textPrimary #1A1917 / textSecondary #6B6862. **다크 전 토큰 존재**(bg #131211 / surface #1C1B1A / text #ECEAE6 …).
- **타이포 스케일**(시스템 폰트): score 34/w700 · titleScreen 22/w700 · **stem 18/w600/1.5(문항 핵심)** · choice 16/w500 · body 15/w400/1.6 · label 12/w600 · caption 13.
- **간격**(4배수): xs4 sm8 md12 lg16 xl24 xxl32, 화면 좌우 패딩 16. **radius**: sm8 md12 lg16 pill999. **elevation**: 평면(그림자 대신 1px outline), 차분함 우선.
- **컴포넌트 세트(이미 존재)**: AppScaffold · QuestionCard · TypeBadge(객관식/OX) · ChoiceTile(idle/pressed/selected/correct/wrong 상태) · AnswerBanner · ExplanationCard · PrimaryButton(하단 고정)/SecondaryButton · ProgressHeader · ScoreView · ModeTile · EmptyState · ThemeRadioGroup · ReviewCard.
> 전체 원문: `docs/UI_DESIGN_SYSTEM.md`.
> **탐색 힌트**(원하면): 폰트 위계·여백 리듬·카드 경계 처리(outline vs 미세 그림자)·정답/오답 피드백 표현·홈의 모드 진입 카드 구성·다크 테마의 깊이감 — 이 지점들에서 더 나은 안을 특히 환영.

## 5. 하드 제약 (반드시 준수 — 위반 시 재작업)

1. **Flutter 재현성**: 최종은 순수 Flutter Material 3 위젯 + ThemeExtension으로 구현한다(외부 UI 패키지 0). CSS로만 가능한 고난도 효과(복잡한 blend, 임의 형태 마스킹 남용)는 피하고 **위젯으로 만들 수 있는 표현**에 머문다.
2. **라이트/다크 동등**: 모든 화면을 **양쪽 테마 모두** 제시. 어느 쪽도 임시방편 아님.
3. **접근성**:
   - 정답/오답은 **색 단독 금지** → 색 + **아이콘(✓/✕)** + 텍스트 3중.
   - 탭 타깃 ≥ 48dp. 명도대비 WCAG AA.
   - 텍스트 스케일 확대(textScaler) 대응 → 고정 높이 대신 최소 높이. 지문이 길어도 깨지지 않게.
4. **콘텐츠가 주인공**: 지문 가독성 최우선. 장식이 콘텐츠를 이기면 안 됨.
5. **레드 절제**: 소방 레드를 **대면적 배경**에 쓰지 말 것(장시간 피로). 액센트로만.
6. **범위**: MVP 5화면만. 통계 대시보드·검색·북마크·온보딩 튜토리얼·소셜 등 **없음**(그리지 말 것).
7. **한국어 UI**. 표시명 "119덱".

## 6. 화면별 디테일 체크리스트 (스펙에서 답해줄 것)

- **홈**: 과목 카드와 모드 4종의 위계(무엇을 가장 크게?), 이어풀기 진입점의 시각적 우선순위, 설정 아이콘 위치.
- **풀이**: 진행바 스타일, 문항 카드 여백/경계, 선택지 idle→selected→correct/wrong **상태별 시각**, 채점 후 정답 하이라이트 방식, 해설 카드 등장 처리, 하단 버튼 고정 레이아웃.
- **OX**: 객관식과 통일 vs 전용 2버튼.
- **시험**: "채점 숨김" 상태에서 선택을 어떻게 표시(중립), 진행 위계, 제출 버튼 전환.
- **결과**: 점수 표현(숫자 위계·정답률), 만점 vs 오답 톤 차이, 오답 리뷰 카드 구성.
- **설정**: 라디오 그룹·앱 정보의 조용한 정리.
- **공통**: 빈 상태("풀 오답이 없어요") 일러스트/아이콘 톤, 에러 상태, 모션(채점 색전환 등) 원칙.

## 7. 산출물 (Deliverables)

### A. 모바일 목업 (HTML/CSS, 인터랙티브)
- 세로 모바일 프레임(권장 375×812 기준, 반응형). 화면별 1개 이상.
- **필수 화면/상태**: 홈(이어풀기 有) · 풀이(채점 전) · 풀이(오답 채점 후, 정답 함께 표시) · 시험(채점 숨김) · 결과(오답 있음) · 설정.
- **라이트/다크 둘 다**. 토글이나 나란히 비교 가능하면 이상적.
- **§8의 실제 문항 카피** 사용.
- 가능하면 선택→채점 색전환 등 **핵심 인터랙션 1~2개**를 살짝 살려라(과하지 않게).

### B. 비주얼 스펙 문서
- 최종 색 토큰(라이트/다크), 타이포 스케일, 간격/모서리, 컴포넌트별 상태 명세(특히 ChoiceTile 전 상태), 모션 원칙.
- baseline(§4)에서 **바꾼 것과 이유**를 명시(탐색 결과이므로).
- Flutter 매핑 힌트(ColorScheme/ThemeExtension 어느 슬롯에 들어갈지) 있으면 가산점.

### 평가 기준 (이 순서로 본다)
1. **가독성·차분함**(장시간 학습에 눈이 편한가) 2. **정답/오답 피드백의 명확함**(색+아이콘, 브랜드 레드와 혼동 없음) 3. **라이트/다크 완성도** 4. **Flutter 구현 현실성** 5. 소방 정체성의 절제된 표현.

## 8. 부록 — 실제 콘텐츠 샘플 (목업에 이 카피를 써라)

과목: **소방관계법규** · 총 25문항(심화 객관식 15 + OX 10).

### 객관식 샘플 (개수형 심화)
> **문항**: 다음 &lt;보기&gt; 중 옳은 것은 모두 몇 개인가?
> ㄱ. (소방기본법) 소방용수시설 조사결과는 2년간 보관한다.
> ㄴ. (위험물법) 이동탱크저장소는 정기점검 대상이지만 예방규정 작성 대상은 아니다.
> ㄷ. (화재예방법) 소방안전관리 전담 대상은 특급·1급·2급 대상물이다.
> ㄹ. (소방시설법) 화재안전기준 강화 시 소화기구·비상경보설비·자동화재탐지설비는 강화된 기준을 적용할 수 있다.
> ㅁ. (화재조사법) 화재조사관은 3명 이상 배치해야 한다.
>
> **선택지**: ① 1개  ② 2개  ③ 3개  ④ 4개  → **정답 ③ 3개**
> **해설**: 옳은 것은 ㄱ·ㄴ·ㄹ (3개). ㄷ 거짓 — 전담 대상은 특급·1급만(2급 제외). ㅁ 거짓 — 화재조사관은 2명 이상.

### OX 샘플
> **문항**: 이동탱크저장소는 정기점검 대상이지만 예방규정 작성 대상은 아니다.
> **선택지**: O / X  → **정답 O**
> **해설**: O. 이동탱크저장소는 정기점검 대상이나 예방규정 작성 대상은 아니다(구분).

### 홈 카피 예시
- 과목: **소방관계법규** (25문항)
- 모드: **전체 풀이** / **랜덤** / **오답 재풀이** / **시험 모드**
- 이어풀기: **정규 · 7/25 이어서 풀기**

### 결과 카피 예시
- 점수: **18 / 25** (정답률 72%)
- 오답 리뷰: "내 답 ② 2개 ✕ · 정답 ③ 3개 ✓"
